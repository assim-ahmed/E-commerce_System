# 📋 E-commerce API - Project Notation

## 🎯 Project Overview
RESTful API for a single-vendor e-commerce platform built with Laravel 11/12, PHP 8.2+, and MySQL.

---

## ✅ Completed Features (as of 06-05-2026)

### 1. Authentication System (100%)
| API | Method | Endpoint | Protection |
|-----|--------|----------|------------|
| Register | POST | `/api/register` | Public |
| Login | POST | `/api/login` | Public |
| Logout | POST | `/api/logout` | Auth + Verified |
| Get Profile | GET | `/api/profile` | Auth + Verified |
| Update Profile | PUT | `/api/profile` | Auth + Verified |
| Resend Verification | POST | `/api/email/verification-notification` | Auth |
| Verify Email | GET | `/api/verify-email/{id}/{hash}` | Signed URL |

**Features:**
- Email verification required before login
- Single session per user (old tokens revoked on new login)
- Role-based (Admin / Customer)
- Admin middleware for protected routes
- Rate limiting: 100 requests/minute

### 2. Categories Management (100%)
| API | Method | Endpoint | Access |
|-----|--------|----------|--------|
| List Categories | GET | `/api/categories` | Public |
| Get Category by ID | GET | `/api/categories/{id}` | Public |
| Get Category by Slug | GET | `/api/categories/slug/{slug}` | Public |
| Create Category | POST | `/api/categories` | Admin only |
| Update Category | PUT | `/api/categories/{id}` | Admin only |
| Delete Category | DELETE | `/api/categories/{id}` | Admin only |

**Features:**
- Auto-generate slug from name if not provided
- Cascade protection (cannot delete category with products)
- Product count included in response

### 3. Brands Management (100%)
| API | Method | Endpoint | Access |

## 📦 API Response Format

All endpoints return consistent JSON:

```json
{
    "success": true|false,
    "data": {},
    "message": "string",
    "errors": {}
}
Success Response (200):
json
{
    "success": true,
    "data": {...},
    "message": "Operation successful"
}
Error Response (4xx/5xx):
json
{
    "success": false,
    "data": null,
    "message": "Error description",
    "errors": {}
}
