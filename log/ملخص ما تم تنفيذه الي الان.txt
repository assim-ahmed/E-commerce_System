🎯 المطلوب: بناء Shopping Cart API (سلة المشتريات)
📋 تفاصيل Cart API المطلوبة
#	الميزة	الوصف
1	إضافة إلى السلة	Guest (cookie) أو Registered User (user_id)
2	عرض السلة	عرض محتويات السلة مع حساب الإجمالي
3	تحديث الكمية	تغيير كمية منتج في السلة
4	حذف من السلة	إزالة منتج من السلة
5	تفريغ السلة	حذف كل المنتجات من السلة
6	تطبيق كوبون	إضافة كوبون خصم على السلة
📊 الجداول المرتبطة بـ Cart
الجدول	الدور
carts	تخزين بيانات السلة (user_id, cookie_id)
cart_items	تخزين منتجات السلة (product_id, variant_id, quantity, price_at_time)
coupons	كوبونات الخصم
products	بيانات المنتجات
product_variants	متغيرات المنتج
🎯 الـ Endpoints المطلوبة لـ Cart API
Method	Endpoint	الوصف	الصلاحية
GET	/api/cart	عرض السلة	Guest/User
POST	/api/cart/items	إضافة منتج إلى السلة	Guest/User
PUT	/api/cart/items/{id}	تحديث كمية منتج في السلة	Guest/User
DELETE	/api/cart/items/{id}	حذف منتج من السلة	Guest/User
DELETE	/api/cart/clear	تفريغ السلة بالكامل	Guest/User
POST	/api/cart/apply-coupon	تطبيق كوبون خصم	Guest/User
DELETE	/api/cart/remove-coupon	إزالة الكوبون	Guest/User
📝 نقاط مهمة يجب مراعاتها في Cart API
النقطة	التفاصيل
نظام المصادقة	استخدام Laravel Sanctum
الزوار (Guest)	استخدام cookie_id المخزن في الـ cookie
المستخدمين المسجلين	استخدام user_id
سعر المنتج	تخزين price_at_time عند الإضافة (لحماية السعر من التغيير لاحقاً)
المخزون	التأكد من توفر المخزون قبل الإضافة
المتغيرات	دعم إضافة منتج بمتغير محدد (لون، مقاس)
نفس المنتج	إذا أضاف نفس المنتج مرتين، يتم زيادة الكمية
الكوبونات	التحقق من صلاحية الكوبون (تاريخ، أقل مبلغ، غير مستخدم)
