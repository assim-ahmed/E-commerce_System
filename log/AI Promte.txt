## 2. وصف المشروع (Project Description)

### **الرؤية العامة:**
بناء REST API احترافي لمتجر إلكتروني متكامل، يعمل كـ backend لتطبيق React منفصل، مع التركيز على:

- **الأداء العالي** (يدعم 100,000+ منتج)
- **الأمان** (صلاحيات محكمة، rate limiting)
- **سهولة التوسع** (Service-Repository Pattern)
- **المرونة** (API قابل للتعديل والتوسع)

### **المستخدمون المستهدفون:**
| المستخدم | الوصف |
|----------|-------|
| **Admin** | مالك المتجر (بائع واحد فقط)، يدير كل شيء |
| **Customer** | المستخدم العادي الذي يشتري المنتجات |

### **الميزات الرئيسية للمشروع:**

| # | الميزة | الوصف |
|---|--------|-------|
| 1 | **نظام المنتجات** | منتجات مع تصنيفات، ماركات، صور، مواصفات |
| 2 | **متغيرات المنتج** | دعم الألوان، المقاسات، السعات بأسعار مختلفة |
| 3 | **سلة المشتريات** | تدعم الزوار (cookie) والمسجلين (user_id) |
| 4 | **الطلبات** | إنشاء طلبات، تعديلها، تتبع حالتها |
| 5 | **الكوبونات** | نظام خصومات بسيط (نسبة مئوية أو قيمة ثابتة) |
| 6 | **التقييمات** | تقييم المنتجات بعد الشراء فقط |
| 7 | **نظام المخزون** | تتبع المخزون مع تحذير عند الوصول للحد الأدنى |
| 8 | **الإشعارات** | إشعارات للمستخدمين (طلبات، مخزون، الخ) |
| 9 | **لوحة تحكم Admin** | إدارة كل شيء من مكان واحد |

⚠️ 6. قواعد مهمة يجب اتباعها عند المساعدة
لا تضيف SoftDeletes لأي Model (لأن قاعدة البيانات لا تحتوي على deleted_at)

لا تغير هيكل قاعدة البيانات من عندك - التغييرات فقط بعد موافقة صاحب المشروع

اتبع العلاقات المحددة في الـ Schema أعلاه

الـ Authentication باستخدام Laravel Sanctum (وليس Passport أو Session)

الـ API Responses يجب أن تكون بصيغة JSON موحدة:

json
{
    "success": true,
    "data": {},
    "message": "",
    "errors": {}
}
دعم الأداء لـ 100,000+ منتج يجب أن يؤخذ في الاعتبار (Indexes, Eager Loading)

الأمان: Rate limiting, Validation, Authorization (Admin/Customer roles)
