# E-commerce API - Authentication System Complete

## 🎯 Project Overview
RESTful API for single-vendor e-commerce platform built with Laravel 11/12, PHP 8.2+, MySQL.

## ✅ Completed Features (Current State)

### Authentication System (100% Complete)

#### 1. User Management
- Users table with fields: id, name, email, password, role (admin/customer), is_active (boolean), email_verified_at, timestamps
- Default role on registration: 'customer'
- Default is_active status: true
- Password hashed using bcrypt

#### 2. API Endpoints (7 Total)
| Method | Endpoint | Description | Middleware |
|--------|----------|-------------|-------------|
| POST | /api/register | Register new customer | None |
| POST | /api/login | Login & get token | None |
| POST | /api/logout | Logout & revoke token | auth:sanctum, verified |
| GET | /api/profile | Get user profile | auth:sanctum, verified |
| PUT | /api/profile | Update profile (name, email, password) | auth:sanctum, verified |
| POST | /api/email/verification-notification | Resend verification email | auth:sanctum |
| GET | /api/verify-email/{id}/{hash} | Verify email | signed URL |

#### 3. Authentication Flow
1. User registers → Creates account with email_verified_at = null
2. System sends verification email
3. User cannot login until email verified
4. User clicks verification link → email_verified_at set to current timestamp
5. User can now login → Receives Bearer token
6. On login, all existing user tokens are deleted (single session only)
7. On logout, current token is revoked

#### 5. Security Features
- Laravel Sanctum for API token authentication
- Email verification required before access to protected routes
- Rate limiting: 100 requests per minute
- Password confirmation validation
- Unique email constraint
- Role-based access (admin middleware available)
- Single session per user (old tokens deleted on new login)

7. Validation Rules
RegisterRequest:

name: required, string, max:255

email: required, email, unique:users

password: required, min:8, confirmed

LoginRequest:

email: required, email

password: required, string

UpdateProfileRequest:

name: sometimes, string, max:255

email: sometimes, email, unique:users (ignore current user)

password: sometimes, min:8, confirmed
